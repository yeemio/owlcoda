Native REPL

Slash commands, tools, sessions, skills, and the training pipeline — everything you do inside owlcoda.

Type owlcoda and you land in the native REPL. It looks like a terminal because it is one.

Sessions

Every conversation is saved to ~/.owlcoda/sessions/<id>.json automatically. To resume:

owlcoda --resume last           # most recent session
owlcoda --resume <id>           # specific id from `owlcoda sessions`

You can tag sessions (/tag launch-blocker), search by tag (owlcoda sessions --tag launch-blocker), and branch from any message (/branch).

Skills (L2)

OwlCoda matches the user’s incoming message against a curated methodology pack plus any user-installed skills under ~/.owlcoda/skills/. Matched skills get injected into the system prompt at request time. Tune behavior:

{
  "skillInjection": true,
  "skillTopK": 2,
  "skillThreshold": 0.1
}

See owlcoda skills info for what’s loaded.

Training pipeline (L3, opt-in / off by default)

{ "trainingCollection": false }

Or runtime override:

OWLCODA_TRAINING_COLLECTION=1 owlcoda

When on, sessions are scored on 5 weighted dimensions, PII-sanitized, and appended to ~/.owlcoda/training/collected.jsonl. Export:

owlcoda training export jsonl --min-quality 70
owlcoda training export sharegpt --tool-calls-only
